# TOOLING CLAUDE REFERENCE

## Overview

The claude stack seeds the `.claude/` workflow directory into a project.

## Structure

```
.claude/
├── CLAUDE.md          ← session system prompt, paste first every session
├── TASKS.md           ← persistent task tracker, source of truth for progress
├── REQUIREMENTS.md    ← project goals, non-goals, MVP scope
├── ARCHITECTURE.md    ← technical design decisions and open questions
├── PROJECT.md         ← auto-generated by bun run snapshot, gitignored
└── FEATURES/
    ├── [feature].md   ← active feature plan, one at a time
    └── done/          ← completed features, never load into context
```

## CLI

| Command              | What it does                                                     |
| -------------------- | ---------------------------------------------------------------- |
| `gdev claude init`   | Seeds `.claude/`, creates `FEATURES/` dirs, updates `.gitignore` |
| `gdev claude update` | Diffs `CLAUDE.md` against seed, offers to apply changes          |
