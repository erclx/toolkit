# TOOLING CLAUDE REFERENCE

## Overview

The claude stack seeds the `.claude/` workflow directory into a project. These documents
support AI-assisted planning sessions with Claude chat. They are project-owned — seeded
once and never overwritten by tooling sync.

## Structure

```
.claude/
├── CLAUDE.md          ← session system prompt, paste first every session
├── TASKS.md           ← persistent task tracker, source of truth for progress
├── REQUIREMENTS.md    ← project goals, non-goals, MVP scope
├── ARCHITECTURE.md    ← technical design decisions and open questions
├── PROJECT.md         ← auto-generated by bun run snapshot, gitignored
└── FEATURES/
    ├── [feature].md   ← active feature plan, one at a time
    └── done/          ← completed features, never load into context
```

## Key Decisions

**Seeds only, never overwritten.** All documents are seeded on `gdev claude init` and
never touched by tooling again. They are project-owned content that evolves with the project.

**CLAUDE.md is the exception.** It has a canonical template that can be updated across
projects via `gdev claude update`, which diffs against the seed and offers to apply changes.

**PROJECT.md is ephemeral.** Generated by `bun run snapshot`, gitignored, never committed.
Regenerate when project structure changes.

## CLI

| Command              | What it does                                                     |
| -------------------- | ---------------------------------------------------------------- |
| `gdev claude init`   | Seeds `.claude/`, creates `FEATURES/` dirs, updates `.gitignore` |
| `gdev claude update` | Diffs `CLAUDE.md` against seed, offers to apply changes          |

## Workflow

See `docs/WORKFLOW.md` for the full session loop and tool mapping.

## Session Setup

Paste into Claude chat in this order every session:

1. `CLAUDE.md` — always first
2. `TASKS.md` — always
3. `FEATURES/[active].md` — always
4. `REQUIREMENTS.md` + `ARCHITECTURE.md` + `PROJECT.md` — new features only
