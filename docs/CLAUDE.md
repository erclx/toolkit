# Claude Workflow

## Overview

The `.claude/` directory holds planning and context documents for AI-assisted development
sessions with Claude chat. These are project-owned files — seeded once by tooling and
evolved manually as the project grows.

## Key Decisions

**Seeds only, never overwritten.** `gdev claude init` seeds the documents once. Tooling
never touches them again. They belong to the project, not the toolkit.

**CLAUDE.md is the exception.** It has a canonical template in the toolkit. Run
`gdev claude update` to diff your project's copy against the seed and apply changes selectively.

**PROJECT.md is ephemeral.** Auto-generated by `bun run snapshot`, gitignored, never
committed. Regenerate when project structure changes meaningfully.

**No governance rules needed in Claude chat.** Rules live in Gemini sessions. Claude chat
is for planning, tracking, and debugging only — keep it lean.

## CLI

| Command              | What it does                                                     |
| -------------------- | ---------------------------------------------------------------- |
| `gdev claude init`   | Seeds `.claude/`, creates `FEATURES/` dirs, updates `.gitignore` |
| `gdev claude update` | Diffs `CLAUDE.md` against seed, offers to apply changes          |
