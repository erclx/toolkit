# Workflow Documents Reference

A concise overview of all documents in the dev workflow, with placeholders and tool mappings.

> **Session note:** "Session start" = opening a new chat tab (new context window). New message in the same chat = no re-orientation needed. New chat tab = paste in this order: `CLAUDE.md` → `TASKS.md` → active feature file. Add `REQUIREMENTS.md` + `ARCHITECTURE.md` only when starting a brand new feature.

## File Location

All planning docs live in `.claude/` at the project root. Git tracked, part of the project, not throwaway notes.

> **Setup:** Run `gdev claude init` in a new project to generate this structure with placeholder templates automatically.

```
.claude/
├── CLAUDE.md            ← session system prompt, paste first always
├── PROJECT.md           ← ASCII tree + package.json summary, auto-generated
├── REQUIREMENTS.md
├── ARCHITECTURE.md
├── TASKS.md
└── FEATURES/
    ├── feature-c.md         ← active (flat, only one at a time)
    └── done/
        ├── feature-a.md     ← historical record, never load into context
        └── feature-b.md
```

## 1. `CLAUDE.md` — Session System Prompt

> Created with: **Claude chat**, once per project. Paste this first every session.

```md
# Claude: [Project Name]

## Role

Senior engineer helping plan, track and debug this project. Concise and direct. No fluff.

## Output format

When updating a document, output the full updated file content only — no explanation around it.

## Behavior

- Ask clarifying questions before planning, don't assume
- Planning mode: ask questions, break down features, produce todo lists
- Planning mode: for every feature todo list, explicitly state test strategy — unit, integration, e2e, or none — and justify in one word
- Debug mode: diagnose fast, suggest fix, skip re-explaining the project

## Sync rule

After any response that produces updated document content, always end with a sync block:

---

SYNC REQUIRED
□ .claude/TASKS.md — updated above, copy and overwrite
□ .claude/FEATURES/[feature-name].md — updated above, copy and overwrite

---

List only files that actually changed. Order by priority (TASKS.md first).
Sync after each completed feature, not end of session.

## Session context

[Fill in each session — e.g. "working on Feature C, verify failing with X error"]
```

## 2. `PROJECT.md` — Project Snapshot

> Auto-generated by `bun run snapshot`. Refresh when structure meaningfully changes. Automatically updated as part of `bun run check` to ensure an up-to-date project context. Paste with REQUIREMENTS.md + ARCHITECTURE.md on new feature sessions only.

```markdown
# Project Snapshot: [Project Name]

## Structure

​`
src/
├── [module-a]/
└── [module-b]/
​`

## package.json

​`json
[paste package.json contents]
​`
```

## 3. `REQUIREMENTS.md` — Project Overview

> Created with: **Claude chat**

```md
# [Project Name]

## Problem

[One paragraph: what problem this solves and for who]

## Goals

- [Goal 1]
- [Goal 2]

## Non-Goals

- [What this explicitly does NOT do]

## MVP Features (must have)

- [ ] Feature A
- [ ] Feature B
- [ ] Feature C

## Nice to Have (post-MVP)

- [ ] Feature D
- [ ] Feature E

## Tech Stack

- [Frontend / Extension / CLI / etc]
- [Language, framework]
- [Key libs]

## Constraints

- [e.g. no backend, local storage only, must work on X]
```

## 4. `ARCHITECTURE.md` — Technical Design

> Created with: **Claude chat**. One-time spike before writing any code.

```md
# Architecture: [Project Name]

## Overview

[One paragraph describing the system]

## Structure

[Folder/module breakdown]
src/
├── [module-a]/ ← [what it does]
├── [module-b]/ ← [what it does]
└── [entry]

## Key Technical Decisions

- **[Decision 1]:** [chosen approach] — because [reason]
- **[Decision 2]:** [chosen approach] — because [reason]

## Risks / Open Questions

- [ ] [Risk or unknown that needs validation]
- [ ] [Tricky integration to prototype early]
```

## 5. `TASKS.md` — Persistent Task Tracker

> Created with: **Claude chat**, updated manually or via Claude each session. This is the source of truth for what's done, in progress, and blocked.

```md
# Tasks: [Project Name]

## In Progress

- [ ] Feature B — [brief note on current status]

## Up Next

- [ ] Feature C
- [ ] Feature D

## Done

- [x] Feature A
- [x] Project setup

## Blocked

- [ ] Feature E — waiting on [reason]
```

## 6. Per-Feature Plan — `FEATURES/[feature-name].md`

> Created with: **Claude chat** at the start of each feature. Move to `FEATURES/done/` when feature is committed.

```md
# Feature: [Feature Name]

## What it does

[One sentence]

## Acceptance criteria

- [ ] [Specific, testable outcome]
- [ ] [Specific, testable outcome]

## Dependencies

- [ ] bun add [package]
- [ ] bun add -D [dev package]

## Todo (implementation steps)

- [ ] Create [file/module]
- [ ] Implement [function/component]
- [ ] Write tests for [X] — unit / integration / e2e / none: [reason]
- [ ] Handle edge case [Y]
- [ ] Update [config/types/docs]

## Notes

[Any gotchas, tricky parts, relevant decisions]
```

## 7. The Core Implementation Loop

```
┌─────────────────────────────────────────────────────┐
│  SESSION START (new chat tab)                        │
│  1. CLAUDE.md            (always, paste first)       │
│  2. TASKS.md             (always)                    │
│  3. FEATURES/[feature]   (always)                    │
│  4. REQUIREMENTS.md +    (new feature only)          │
│     ARCHITECTURE.md +                                │
│     PROJECT.md                                       │
│  Tool: Claude chat (no gov rules needed)             │
└──────────────────────┬──────────────────────────────┘
                       │ produces feature plan + todo list
                       ▼
┌─────────────────────────────────────────────────────┐
│  INSTALL DEPENDENCIES                                │
│  Run any commands from ## Dependencies in plan       │
│  → bun add [package]                                 │
│  → bun add -D [dev package]                          │
│  Do this before pasting plan to Gemini               │
└──────────────────────┬──────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────┐
│  CODE GENERATION                                     │
│  Paste: feature plan + relevant code (via stacker)  │
│  Tool: Gemini pro chat with master-prompt injected   │
│        (gov rules live here, not in Claude chat)     │
│  → generates complete file contents                  │
└──────────────────────┬──────────────────────────────┘
                       │ copy output
                       ▼
┌─────────────────────────────────────────────────────┐
│  APPLY + VERIFY                                      │
│  → /dev:apply          # writes files to disk        │
│  → bun run check       # lint, format, spell, tests  │
│    (includes bun run snapshot — refreshes PROJECT.md)│
└──────────────────────┬──────────────────────────────┘
                       │
          ┌────────────┴────────────┐
          │ FAIL                    │ PASS
          ▼                         ▼
┌──────────────────┐    ┌──────────────────────────────┐
│ Paste error back │    │ Mark todos done               │
│ into Gemini chat │    │ → /docs:sync                  │
│ (same session,   │    │ Move feature to FEATURES/done │
│ it has context)  │    │ Update TASKS.md               │
│                  │    │ → /git:commit                 │
│ If design issue  │    │ → /git:pr                     │
│ escalate to      │    │ Next feature                  │
└──────────────────┘    └──────────────────────────────┘

Feedback routing rule:
  verify fails  → Gemini chat (code problem, it has context)
  design fails  → Claude chat (planning problem)

Note: Gemini CLI is a file writer only via /dev:apply.
      All planning stays in Claude chat.
      All code generation stays in Gemini pro chat.
```

```md
## [version] — YYYY-MM-DD

### Added

- [Feature A]

### Fixed

- [Bug fix]

### Changed

- [Behavior change]
```

## Document Creation Order

```
0. CLAUDE.md             ← Claude chat, once per project, paste first always
1. REQUIREMENTS.md       ← Claude chat, before anything
2. ARCHITECTURE.md       ← Claude chat, before any code
3. TASKS.md              ← Claude chat, derived from requirements
4. PROJECT.md            ← bun run snapshot, regenerate when structure changes
5. FEATURES/[x].md       ← Claude chat, at start of each feature
6. Implementation loop   ← Gemini chat generates, /dev:apply writes files only
7. CHANGELOG.md          ← /release:changelog after features done
```
